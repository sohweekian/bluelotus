However, Risk Challenger in that cycle has:

  confidence: 0.5
  retry_count: 2

File:

  C:\bluelotus3\data\v3_cycles\v3_cycle_20260621_014257\agent_reports\risk_challenger.json

This means the exact scaffold-empty bug was fixed, but a neighboring "no
findings" pathway remains.

Root cause:

  The semantic gate only fires when all three conditions are true:

    not summary
    not key_findings
    confidence == 0.5

  In cycle 014257, Risk Challenger had a non-empty summary, so the gate allowed
  the response even though key_findings was empty and confidence remained 0.5.

Code locations:

  C:\bluelotus3\agents\base_agent.py lines 67, 99, 154, 186

Impact:

  If v3_cycle_20260621_014257 is published, the Chief Strategist page can again
  show "No findings" for Risk Challenger.

7. RECOMMENDED FOLLOW-UP
-------------------------------------------------------------------------------
Recommended fix:

  Replace the narrow semantic-empty check with a semantic adequacy gate.

Suggested gate:

  - key_findings must contain at least 1 non-empty string for every accepted
    agent report.
  - summary must be non-empty.
  - if confidence == 0.5 and key_findings is empty, trigger retry.
  - after max retries, either:
      A. raise agent error and let degraded card render, or
      B. synthesize a deterministic degraded agent report explicitly marked
         "INSUFFICIENT_AGENT_CONTENT".

Recommended tests:

  - test_semantic_gate_retries_scaffold_empty_response
  - test_semantic_gate_retries_nonempty_summary_but_empty_findings
  - test_max_retry_empty_findings_does_not_publish_as_normal_agent
  - test_public_v3_agents_contract_requires_nonempty_key_findings
