3. HYPOTHESES
────────────────────────────────────────────────────────────────────────────────

Bug A Hypothesis:
  qwen3:4b interprets "Return this shape exactly" literally. It fills in
  metadata fields it can copy from the prompt (cycle_id, agent_id, timestamps)
  and returns the content arrays verbatim from the scaffold template.
  The response is schema-valid → result["ok"]=True → retry loop exits at 0.
  This is a semantic emptiness failure: structurally valid, analytically hollow.

Bug B Hypothesis:
  qwen3:4b generates a detailed macro analysis response that exceeds the 800
  token output cap (num_predict: 800 in model_registry.yaml). The Ollama
  server stops generation at the token limit, producing a truncated JSON string
  that fails parsing. Retries also truncate at the same limit.


────────────────────────────────────────────────────────────────────────────────
4. FILES CHANGED
────────────────────────────────────────────────────────────────────────────────

Fix A — agents/base_agent.py
  Added semantic content gate in BOTH run_legacy_prompt_path() and
  run_new_prompt_architecture(), applied after each LLM call (initial + retries).

  Gate logic:
    if result["ok"] and not summary and not key_findings and confidence == 0.5:
        result = {"ok": False, "error": "EMPTY_CONTENT_RESPONSE: ...", ...}

  This feeds the existing retry loop without restructuring it. After the gate
  fires, the retry loop runs the EMPTY_CONTENT_RESPONSE repair prompt, which
  explicitly tells the model to populate findings from the evidence packet.

Fix A — prompting/retry_prompt_builder.py
  Added EMPTY_CONTENT_RESPONSE handling:
  - classify_error(): detects "empty_content_response" in error string
  - _REPAIR_INSTRUCTIONS: repair instruction demanding at least 1 finding
  - build_retry_user_prompt(): dedicated case that shows the agent its previous
    empty response and explicitly demands real content
  - should_retry(): EMPTY_CONTENT_RESPONSE explicitly marked as retryable
    (before the existing "empty" substring block)

Fix B — config/model_registry.yaml
  num_predict raised from 800 to 1400 for default_specialist model role.
  This gives qwen3:4b 75% more output budget, preventing JSON truncation
  on verbose agents like macro_strategist.
