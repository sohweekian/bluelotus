==============================================================================
0. EXECUTIVE SUMMARY
==============================================================================

Build and integrate a new V3 pipeline layer:

    CHIEF STRATEGIST GOVERNANCE LAYER (CSG)

Purpose:

    Prevent inconsistent Chief Strategist communication by forcing every
    strategist briefing to reconcile:

        1. CIO Letter / institutional memory
        2. Master Operating Prompt / execution doctrine
        3. active thesis registry
        4. current event-to-thesis mapping
        5. tactical dataset / market tape
        6. portfolio and open orders
        7. kill conditions
        8. strategic-vs-tactical reconciliation
        9. allowed / forbidden interpretations

before any Chief Strategist narrative is generated.

The failure mode to solve:

    The Chief Strategist treated a tactical score such as
    "Gold Thesis WATCH 0.50" as if it could override or diminish the
    structural inflation thesis recorded in the CIO Letter and institutional
    archive.

Correct doctrine:

    Tactical score modifies timing.
    Tactical score does not invalidate structural thesis unless the
    defined kill condition is triggered.

This work order adds a mandatory machine-readable context object into
`dataset_raw.json`, surfaces the same governance logic in TXT / Word / Excel
reports, and creates a post-generation audit gate so future Chief Strategist
replies remain fluid, memory-aware, and source-grounded across new
conversations, even when only partial intelligence packages are uploaded.

==============================================================================
3. DESIGN PRINCIPLES
==============================================================================

3.1 Hard Source Priority

Every Chief Strategist reply must observe this hierarchy:

    1. CIO Letter / Website Archive
    2. Master Operating Prompt
    3. Thesis Registry
    4. Current Event Map
    5. Tactical Dataset / Report
    6. Portfolio / Orders / Execution Intelligence
    7. Chief Strategist Inference

No tactical score may override a higher-order structural thesis unless the
registered kill condition is triggered.

3.2 Strategic vs Tactical Separation

Every active thesis must have two independent fields:

    strategic_status
    tactical_status

Examples:

    strategic_status = ACTIVE_BULLISH
    tactical_status  = WATCH_MIXED

Correct reconciled label:

    ACTIVE_BUT_TACTICAL_CONFIRMATION_MIXED

Incorrect label:

    WEAK / INVALID / NO BUY

3.3 Chief Strategist Cannot Answer From Raw Tape Alone

The strategist generator must be blocked unless CSG pack is present.

3.4 Partial Package Robustness

Future conversations may receive only:

    - dataset_raw.json
    - or Bluelotus_V3_Report.txt
    - or Word report
    - or Excel report
    - or screenshots
    - or partial package

The system must still provide doctrine-aware replies by embedding the CSG
context inside every major output artifact.

3.5 CIO Execution Safety Remains Unchanged

The CSG layer must not generate orders.

Hard constants:

    CIO_ONLY_MANUAL = TRUE
    ORDER_ROUTING_ENABLED = FALSE
    LLM_ORDER_GENERATION_ENABLED = FALSE
    SYSTEM_GENERATED_ORDERS = 0

CSG is interpretation governance only.

==============================================================================

Create:

    run_chief_strategist_reply_audit.py

Purpose:

    Validate final Chief Strategist draft before publication.

Audit must fail if the draft:

    1. Mentions gold tactical WATCH/FAIL without explaining structural gold thesis.
    2. Mentions Iran de-escalation as gold-bearish without saying structural inflation thesis remains.
    3. Mentions banks without linking to NIM, credit stress, curve, and BAC/WFC/XLF behavior.
    4. Confuses scout with second tranche.
    5. Treats satellites as core holdings.
    6. Omits kill conditions.
    7. Omits probability/confidence bands when making forecasts.
    8. Omits current event linkage.
    9. Contradicts deterministic blocked actions.
    10. Violates CIO_ONLY_MANUAL doctrine.

Audit output:

{
  "reply_audit_version": "v3.5-csg-audit-001",
