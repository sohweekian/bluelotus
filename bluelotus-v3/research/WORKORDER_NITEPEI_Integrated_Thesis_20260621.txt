PROBLEM
-------
The BlueLotus NITE-PEI Integrated Thesis has been written and approved by
CIO Soh Wee Kian and Dr. Claude Code. The thesis defines a 7-sub-engine,
20-step deterministic workflow for:

  (1) Classifying incoming news events against active theses
  (2) Updating thesis probability via Bayesian inference (LR × Prior Odds)
  (3) Tracking kill-condition states (INACTIVE → WATCH → TRIGGERED → CONFIRMED)
  (4) Computing Composite Kill Risk Index (CKRI) replacing binary cash_fortress_mode
  (5) Coupling NITE-PEI posterior probability to Kelly ESM position sizing
  (6) Gating Kelly fraction by signal entropy (SEM) and branch dispersion (PEI)
  (7) Calibrating the Likelihood Ratio Table via the Brier loop post-resolution
  (8) Writing thesis probability as a canonical field in v3_agents_latest.json
  (9) Enforcing dual action gates: equity_action_gate vs. hedge_action_gate

None of these 9 capabilities exist in code. The thesis is a specification only.
The CIO cannot use a thesis. The CIO can only use running software.

This Work Order converts the thesis into running code, in 6 phases, per the
implementation roadmap defined in thesis §19–§24.


ROOT CAUSE (WHY THIS GAP EXISTS)
---------------------------------
Prior to this session, BlueLotus V3 handled thesis probability as:
  - Qualitative narrative ("Gold Thesis WATCH")
  - No Bayesian update formula
  - No quantified kill condition probability
  - No live coupling between thesis state and Kelly position sizing
  - Binary cash_fortress_mode flag (no continuous risk index)
  - VXX/VIXY governance by narrative doctrine only (no code enforcement)

The NITE-PEI thesis closes all six gaps. This Work Order builds the code.
