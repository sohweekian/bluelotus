===============================================================================
3. THESIS

For a normal BlueLotus V3 agent report to be publishable, schema validity is not
enough.

A report must satisfy a minimum semantic content contract:

  1. summary must be non-empty.
  2. key_findings must be a list.
  3. key_findings must contain at least one non-empty finding.

If this contract fails, the report should not be allowed into the published V3
Chief Strategist Council as a normal validated agent report. It must either be
retried or degraded explicitly.

===============================================================================
4. HYPOTHESIS

If the V3 agent runner rejects reports with empty or missing key_findings, not
only fully blank scaffold reports, then Qwen3:4B will be forced through the
existing retry mechanism to produce at least one concrete finding per desk.

Expected result:

  - Empty summary is rejected.
  - Missing/non-list key_findings is rejected.
  - Empty key_findings is rejected even when summary is non-empty.
  - Scaffold default attempts are preserved as failed-attempt evidence.
  - Successful retries produce real findings.
  - Website publishes 9 agents with no empty findings.

5. CHANGES IMPLEMENTED

Modified:
  C:\bluelotus3\agents\base_agent.py

Added helper:

  semantic_content_error(report)

The helper returns True when:

  - summary is blank, or
  - key_findings is not a list, or
  - key_findings has no non-empty item.

Then all four semantic gate locations were changed from the old narrow test:

  summary empty AND key_findings empty AND confidence == 0.5

to the new stronger contract:

  semantic_content_error(report)

Modified:
  C:\bluelotus3\prompting\retry_prompt_builder.py

Retry prompt now explicitly states:

  "Minimum content contract: summary must be non-empty and key_findings must
  contain at least one non-empty item."

JSON repair retries now also instruct the model that key_findings must contain at
least one concise string. If evidence cannot be reconstructed, it must emit a
CIO-verification finding instead of returning an empty array.

Added:
  C:\bluelotus3\tests\test_agent_semantic_content_gate.py

This new regression test suite covers:

  - Blank scaffold response.
  - Non-empty summary with empty key_findings.
  - Blank-string findings.
  - Valid summary plus one finding.
  - Retry prompt contract for EMPTY_CONTENT_RESPONSE.
