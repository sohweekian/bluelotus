2. THESIS
-------------------------------------------------------------------------------
Schema-valid output is not sufficient for institutional agent reporting.

For a BlueLotus V3 agent report to be publishable as a normal agent report, it
must contain at least:

  - a non-empty summary, and
  - at least one non-empty key_findings item.

If an agent cannot produce a key finding after retries, it should not be silently
published as a normal agent. It should either:

  A. retry with a semantic adequacy repair prompt, or
  B. fail the agent and allow the existing degraded/agent-error flow to expose
     the weakness.

3. HYPOTHESIS
-------------------------------------------------------------------------------
The current gate is too narrow because it triggers only when:

  summary is empty AND key_findings is empty AND confidence == 0.5

This misses cases where Qwen fills a generic summary but leaves key_findings
empty.

If the semantic gate is changed to reject:

  - blank summary, OR
  - empty/non-string key_findings, OR
  - only blank key_findings items,

then no normal accepted agent report should reach the publisher with
"No findings" or equivalent empty findings.
