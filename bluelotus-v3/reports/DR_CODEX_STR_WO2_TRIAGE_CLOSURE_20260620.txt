2. Defect 1 - STR-5 Hedge Ratio Review
--------------------------------------
Status: RESOLVED

Root cause:
hedge_ratio_reviewer.py read VXX/VIXY only through a narrow market_value alias path.
The live portfolio stores the field as mkt_val, so VXX and VIXY were matched by ticker but their
market value resolved to 0.0.

Fix:
- Added portfolio position normalization for dict and list shapes.
- Added market value alias resolution: mkt_val, market_val, market_value, value, position_value.
- Added qty x price fallback.
- Added explicit output audit fields:
  hedge_value_source
  hedge_tickers_expected
  hedge_tickers_matched

3. Defect 2 - STR-4 Kelly Advisory Exact Zeroes
-----------------------------------------------
Status: RESOLVED

Root cause:
kelly_edge_calculator.py treated analyst upside <= 0 as a non-computable Kelly condition and
silently returned full_kelly_fraction = 0.0000 under KELLY_NO_SIZE. This made negative available
inputs visually indistinguishable from a missing-input default.

Fix:
- Added explicit Kelly input extraction and audit fields:
  kelly_b, kelly_p, kelly_q, kelly_input_status, kelly_missing_inputs.
- Added KELLY_INSUFFICIENT_DATA / CIO_REVIEW_REQUIRED for missing/null/zero required inputs.
- Negative analyst upside now becomes NEGATIVE_EXPECTED_RETURN and a non-zero negative-edge value,
  not a silent 0.0000 default.

Live-cycle AMD confirmation:
