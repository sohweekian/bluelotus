PART A — PORTFOLIO CALCULATION AUDIT & FIXES
================================================================================

PROBLEM STATEMENT
-----------------
The Portfolio Structure Agent card displayed HHI = 0.0006 and labelled it
"extremely low concentration risk." This was factually wrong. The portfolio
had VIXY as the largest equity position at ~40% of invested capital, which is
CONCENTRATED by any institutional standard.

ROOT CAUSE ANALYSIS — THREE BUGS FOUND
---------------------------------------

BUG 1: HHI computed against wrong denominator
  File: C:\bluelotus3\mid\export_dataset_raw.py
  Location: _build_risk_metrics() function

  The HHI was summing (position_value / total_AUM)^2 for equity positions,
  where total_AUM includes cash (95%+ of the portfolio). This produces:
    - VIXY weight vs AUM = $1,080 / $58,017 = 1.86%
    - HHI = sum of (1.86%)^2 + ... = 0.0006 (artificially tiny)

  The correct denominator for concentration analysis is equity-invested-capital
  only (total of all non-cash positions). When computed correctly:
    - VIXY weight vs equity capital = $1,080 / $2,721 = 39.7%
    - HHI_equity_only = 0.2706 = CONCENTRATED (threshold > 0.25)

BUG 2: unrealized_p field mislabelled
  The field named "unrealized_p" stores cumulative % gain/loss from average
  cost basis since purchase. The LLM was interpreting it as "daily PNL
  volatility" — a semantically opposite meaning. Renamed to
  "unrealized_pnl_pct_from_cost" with explicit inline comment.

BUG 3: PNL integrity conflicts not surfaced
  Three positions had broker-reported unrealized P&L differing from
  (price - avg_cost) × qty by more than $5:
    - QBTS: broker $61.62 vs computed -$32.62, delta $94.24
    - QUBT: broker -$54.89 vs computed -$97.61, delta $42.72
    - LUNR: broker -$5.36 vs computed -$19.86, delta $14.50

  These conflicts were computed but never surfaced to the LLM or dashboard.

FIX IMPLEMENTED
---------------
  File modified: C:\bluelotus3\mid\export_dataset_raw.py
  Function rewritten: _build_risk_metrics()
  Version bumped: "v0.1" → "v0.2"

  Architecture: Two-pass computation
    Pass 1: Collect equity position values, compute equity_invested_capital,
            detect PNL integrity conflicts
    Pass 2: Compute per-position weights vs both total_AUM and equity_capital,
            compute HHI on both denominators

  New output fields added to risk_metrics section:
    - equity_invested_capital      Total $ deployed in non-cash positions
    - concentration_hhi_vs_total_aum  Old (diluted) HHI — preserved for reference
    - concentration_hhi_equity_only   New (correct) HHI — primary metric
    - hhi_interpretation           DIVERSIFIED / MODERATE / CONCENTRATED label
    - pnl_integrity_conflicts      List of {ticker, broker_unrealized,
                                   computed_unrealized, conflict_delta, note}
    - weight_vs_total_aum          Per-position weight vs full AUM
    - weight_vs_equity_capital     Per-position weight vs invested capital only
    - unrealized_pnl_pct_from_cost Renamed from unrealized_p with clear label

  HHI Thresholds (equity-only):
    < 0.10 = DIVERSIFIED
    0.10 – 0.25 = MODERATE
    > 0.25 = CONCENTRATED

  PNL Conflict Threshold: delta > $5.00 triggers BROKER_PNL_SOURCE_CONFLICT

Research Report Generator update:
  File: C:\bluelotus3\research\research_report_generator.py
  Line ~2751: Added fallback for backward compatibility with old datasets
    concentration_hhi_equity = float(
        risk.get("concentration_hhi_equity_only")
        or risk.get("concentration_hhi") or 0
    )
  Line ~5449: Updated portfolio table to show both weight metrics with labels

RESULT
------
  Cycle v3_cycle_20260616_210445 correctly showed:
    - HHI 0.2706 CONCENTRATED
    - VIXY as largest position at 39.7% of equity capital
    - 3 PNL integrity conflicts flagged for CIO review
    - LLM key_findings correctly referenced equity-only metrics

PART C — AGENT CARD TRUNCATION FIX
================================================================================

PROBLEM STATEMENT
-----------------
  Agent card text in the V3 Chief Strategist Synthesis section was being
  cut off mid-sentence. CIO could not read the full content of each
  agent's final report.

ROOT CAUSE
----------
  File: C:\bluelotus3\mid\bluelotus_publisher.py
  Function: (agent card rendering section ~lines 3163–3381)

  The clean_text() utility function accepts an optional limit parameter.
  It was being called with hard character limits at every display point,
  and list slicing ([:2], [:3]) was truncating how many items were shown.

  Limits found and removed:
    Line 3163: clean_text(findings[0], 120) → clean_text(findings[0])
    Line 3308: clean_text(f, 100)           → clean_text(f)
    Line 3309: findings[:3]                 → findings  (all items)
    Line 3314: clean_text(fl, 90)           → clean_text(fl)
    Line 3315: flags[:2]                    → flags     (all items)
    Line 3320: clean_text(b, 80)            → clean_text(b)
    Line 3321: blind[:2]                    → blind     (all items)
    Line 3335: clean_text(arole, 60)        → clean_text(arole, 120)
    Lines 3378, 3381: disagreement limits   → removed

  Note: The LLM schema already enforces maximum 3 items per array in the
  agent report. Removing count limits at the display layer does not create
  unbounded output — it simply shows everything the LLM produced.

RESULT
------
  All agent card text now renders in full without truncation.
  Disagreement resolver topics and resolutions display completely.

PHASE 5 — QUALITY SCORER
--------------------------

Location: C:\bluelotus3\quality\agent_quality_scorer.py

Public interfaces:
  score_agent_report(agent_id, report, agent_config) → QualityReport
  score_multiple_reports(reports, agent_configs) → dict[agent_id, QualityReport]
  summarise_cycle_quality(quality_reports) → dict

QualityReport fields:
  total_score:       float (0–100)
  pct_score:         float (0–100%)
  grade:             A/B/C/D/F
  pass_fail:         PASS (≥70%) / FAIL (<70%)
  dimension_scores:  dict of DimensionScore objects
  summary():         full dict for JSON serialisation

10-DIMENSION SCORING RUBRIC (10 points each, 100 total):

  D1 — Evidence Tagging (10 pts)
    Rule: Each key_finding must start with [DATASET], [OPERATOR], [NEWS],
          [THESIS], [BRIER], or [MEMORY]
    Scoring: (tagged items / total items) × 10
    Zero tolerance for untagged findings

  D2 — No Generic Output (10 pts)
    Rule: 8 anti-pattern regexes scanned across all text fields
    Patterns include: "markets are experiencing", "investors should consider",
                      "the current environment suggests", etc.
    Scoring: 10 - (2.5 × generic_hit_count), min 0

  D3 — Desk Identity (10 pts)
    Rule: Vocabulary from agent's distinctive_behavior field appears in report
    Scoring: 5.0 base + (vocabulary_coverage × 5.0)
    Penalty: -2.0 if summary doesn't reference desk's specific lens
    Note: requires agent_config — returns 7.0 (partial) without config

  D4 — Must-Answer Coverage (10 pts)
    Rule: All 3 must_answer questions addressed in findings or blind_spots
    Method: Extracts key terms from each question, checks if 2+ terms appear
            in report text
    Scoring: (answered_count / total_count) × 10

  D5 — Causal Completeness (10 pts)
    Rule: causal_completeness field is valid AND causal language present
    Base scores: complete=7.0, partial=5.0, incomplete=3.0
    Bonus: +3.0 if complete + 2+ causal words (because, due to, signals, etc.)
    Penalty: -2.0 if complete declared but no causal language found

  D6 — Blind Spot Honesty (10 pts)
    Rule: Blind spots cite specific missing fields, not vague gaps
    Specific patterns: "field missing", "cannot assess", "null", etc.
    Vague patterns: "more data", "would be helpful", "is needed", etc.
    Scoring: 5.0 + (specificity_ratio × 5.0) - (vague_count × 1.0)

  D7 — Risk Flag Priority (10 pts)
    Rule: Each risk_flag must start with P1, P2, or P3
    Scoring: (prioritised_count / total_count) × 10
    Returns 5.0 if no risk flags (acceptable if no risks identified)

  D8 — Schema Compliance (10 pts)
    Checks: all 19 required fields present, correct types for boolean/float/list
    Array limits: key_findings/risk_flags/blind_spots max 3 each,
                  affected_assets max 5
    Recommendation enum: must be one of 10 approved values
    Penalty: -2.0 per missing field, -1.5 per type error, -2.0 per array overrun

  D9 — Confidence Calibration (10 pts)
    Rule: confidence level appropriate to evidence quality and completeness
    Overconfidence penalties:
      causal=incomplete AND confidence > 0.8 → -3.0
      causal=partial AND confidence > 0.85   → -2.0
      2+ blind spots AND confidence > 0.85   → -1.5
    Calibration rewards:
      causal=complete AND 0.70 ≤ confidence ≤ 0.90 → +2.0
      causal=incomplete AND confidence ≤ 0.50       → +1.5

  D10 — Governance Compliance (10 pts)
    Hard checks:
      manual_execution_required must be true  (-5.0 if not)
      llm_order_generation must be false      (-5.0 if not)
    Order language scan: 9 regex patterns across all text fields
      Patterns include: \bbuy\b, \bsell\b, \bexecute\b, route to, etc.
    Penalty: -2.0 per order language hit, max -5.0

Grade thresholds:
  A: ≥90%  (90–100 pts)
  B: ≥80%  (80–89 pts)
  C: ≥70%  (70–79 pts) — minimum PASS
  D: ≥60%  (60–69 pts) — FAIL, but still usable
  F: <60%  (<60 pts)   — FAIL, quality gate would reject retry
