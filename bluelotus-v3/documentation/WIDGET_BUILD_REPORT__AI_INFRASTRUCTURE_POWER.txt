SECTION 5 — SCORING LOGIC
────────────────────────────────────────────────────────────────────────────────

5.1  TICKER SIGNAL CLASSIFICATION

    For each ticker, compare day_change_pct to thresholds from config:
      day_change_pct >= pass_pct (+0.5%)   →  PASS
      day_change_pct <= fail_pct (-0.5%)   →  FAIL  (inclusive boundary)
      between                              →  WATCH
      data unavailable                     →  UNKNOWN

5.2  BASKET SCORING

    basket_score = (PASS_count / total_enabled_tickers) * basket_weight
    basket_signal = PASS  if >50% tickers PASS
                    FAIL  if >50% tickers FAIL
                    WATCH otherwise

5.3  HEADLINE SCORING

    Reads data/headlines_live.json (graceful degradation if file absent).
    Scans all headlines across all sources for 25 config-defined keywords.
    headline_score = (unique_keywords_matched / total_keywords) * headline_max
    Capped at headline_max (15 pts).

5.4  RISK PENALTY

    IF SPY day_change_pct < spy_threshold
    AND QQQ day_change_pct < qqq_threshold:
        penalty = min(risk_penalty_max, |avg(SPY, QQQ)| * multiplier)
    Penalty is subtracted from total score.
    Only triggers when BOTH benchmarks breach simultaneously.

5.5  TOTAL SCORE AND STATUS RESOLUTION

    total = basket_total + headline_score - risk_penalty

    STATUS THRESHOLDS (from config — not hardcoded):
      total >= 75  →  CONFIRMING
      total >= 55  →  WATCH
      total >= 40  →  MIXED
      total >= 20  →  WEAKENING
      total <  20  →  CONTRADICTED
      no data      →  UNKNOWN

5.6  CONFIDENCE RESOLUTION

    data_fraction = available_tickers / total_thesis_tickers
      >= 70%  →  HIGH
      >= 45%  →  MEDIUM
      >= 20%  →  LOW
      <  20%  →  UNKNOWN

5.7  CIO ACTION RESOLUTION

    Key = status + "_" + confidence  (e.g. "CONFIRMING_HIGH")
    Looked up in cio_action_map from YAML config.

    Allowed CIO action values:
      WAIT | HOLD | HOLD_REVIEW | NO_ADD | RISK_REVIEW | CIO_REVIEW_REQUIRED

    Forbidden: BUY, SELL, SHORT, COVER, EXECUTE, ORDER — none appear anywhere
    in the system. Verified by test T8.6 (no_buy_sell_in_cio_actions).

11.1  KNOWN LIMITATIONS

    L1 — NO MARKET-HOURS DETECTION
         When US markets are closed, yfinance returns the previous session's
         closing prices. The widget does not flag "MARKET CLOSED — showing
         last session close." Day_change_pct during closed hours reflects
         the last trading day, not an intraday move.
