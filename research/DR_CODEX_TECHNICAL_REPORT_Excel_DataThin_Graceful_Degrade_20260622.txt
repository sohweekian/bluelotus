3. WHY THIS HAPPENED ON A WEEKEND
──────────────────────────────────
The ACMS_NITE_News_Recon sheet holds the Chief Clerk / Contradiction Mapper's
reconciliation table: one row per detected contradiction across ACMS, NITE-PEI,
and News sources.

On a regular trading day, the Clerk maps ≥5 contradictions (between agent signals,
thesis probabilities, news classifications, and portfolio state). The minimum of 6
(header + ≥5 data rows) was calibrated against normal session throughput.

On a weekend / closed-market cycle:
  - No intraday price action → fewer signal-vs-signal conflicts
  - No fresh order flow → fewer execution contradictions
  - Market session_flag = CLOSED → some agent categories produce minimal output
  → Clerk genuinely and correctly generates fewer contradictions (4 this cycle)
  → This is not a data quality failure — it is a correct reflection of reality

The defect is that the validation treated a thin-but-correct dataset as a hard
failure and aborted the entire 30+ sheet workbook as a result.


4. ARCHITECTURAL FLAW IDENTIFIED
──────────────────────────────────
The single-sheet row count guard (_validate_acms_nite_news_rows) used RuntimeError
as its failure mode. This caused a cascade:

    ONE THIN SHEET → ENTIRE WORKBOOK ABORTED

This is disproportionate. A thin reconciliation table on a quiet weekend session
is useful information — it should appear in the Excel with a DATA_THIN marker,
not cause the CIO to receive no Excel at all.

Contrast:
  - TXT: renders whatever it receives, no gate
  - DOCX: renders whatever it receives, no gate
  - XLSX: hard gate → all-or-nothing failure

This asymmetry means the most structured output (XLSX, which the CIO actually
opens for column-level analysis) is the one most likely to be missing.


5. FIX APPLIED
───────────────
File: C:\bluelotus3\research\research_report_generator.py

A. NEW FUNCTION — _validate_and_pad_acms_nite_news_rows()
   Replaces the raising behavior with warn-and-pad:

   for each sheet below minimum:
       → append DATA_THIN sentinel rows until minimum is met
       → record warning string in warnings_out list
       → DO NOT raise — execution continues

   DATA_THIN sentinel row format:
       col[0] = "DATA_THIN"
       col[1] = "Thin-cycle placeholder — <full warning message>"
       col[2..n] = "" (empty, ncols derived from header row)

   Returns: List[str] of warning messages (empty if all sheets are healthy)

B. CALL SITE UPDATE — build_excel_report() at line ~6378
   Before:
       _ann_rows = _acms_nite_news_excel_rows(dataset)
       _validate_acms_nite_news_rows(_ann_rows, dataset)   # raises on thin

   After:
       _ann_rows = _acms_nite_news_excel_rows(dataset)
       _ann_thin_warnings = _validate_and_pad_acms_nite_news_rows(_ann_rows, dataset)
       for _w in _ann_thin_warnings:
           print(f"Excel DATA_THIN warning: {_w}")           # logged, not raised

C. BACKWARD COMPAT SHIM — _validate_acms_nite_news_rows() preserved
   Now delegates to _validate_and_pad_acms_nite_news_rows() and discards return
   value. Any external caller (test files, future scripts) calling the old name
   will still work without modification.

7. DATA_THIN MARKER BEHAVIOUR IN EXCEL
────────────────────────────────────────
When a thin-cycle pad row appears in the workbook:
  - Cell A = "DATA_THIN" (visible, searchable)
  - Cell B = full warning text with session context and row counts
  - Remaining cells = empty
  - Row styling: follows existing ACMS_NITE_News_Recon style logic
    (P1/CRITICAL → red, else amber) — DATA_THIN will be amber (non-critical)

The CIO can immediately see that the sheet has fewer real contradictions than
the normal minimum, with the session context (CLOSED / WEEKEND) inline.

10. CODEX REFERENCE NOTES
──────────────────────────
DO NOT revert to raising RuntimeError in _validate_acms_nite_news_rows.
The old approach was an over-correction from an earlier defect where empty
sheets went undetected. The fix path is warn+pad, not abort.

If minimum thresholds need to be adjusted per session type (e.g. CLOSED vs OPEN),
extend the function signature to accept a session_flag parameter and use a
session-aware minimum table. Do not re-introduce unconditional raises.

The _validate_acms_nite_news_rows() shim must remain in place until confirmed
that no external test or pipeline script calls it by the old name.

If a future defect requires identifying which cycles had thin data, query
the delivery JSON warnings[] array — every thin-cycle warning is recorded there
with full session context and timestamp.
